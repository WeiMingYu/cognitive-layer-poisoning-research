**CLPMAI Paper 2**

**跨層攻擊鏈：從供應鏈投毒到 AI Agent 執行**

草稿章節 --- Layer 1：供應鏈攻擊面

**Layer 1：供應鏈攻擊面------AI 生態系中的三條污染管道**

**1.1 概覽與定位**

Layer 0 描述了使攻擊得以可行的結構性先決條件------使用者信任、API Key
耦合，以及監控缺口。Layer 1
是攻擊鏈的第一個主動層，描述攻擊者如何在技術基礎設施層面植入污染，使後續的認知漂移得以發生。

Layer 1 的核心特徵是：污染發生在開發者的工作流程上游，在任何 AI
輸出被使用者接收之前就已完成。這使得 Layer 1
攻擊具有持久性------一旦污染進入供應鏈，所有下游使用者均受影響，且無需對每個受害者個別施加攻擊。

與傳統軟體供應鏈攻擊（如 SolarWinds、XZ Utils）相比，Layer 1 的 AI
供應鏈攻擊具有三個顯著差異：

- 污染對象是模型的語意輸出，而非程式碼的執行行為------現有程式碼審計工具完全無法偵測。

- 污染效果是概率性的，而非確定性的------受污染模型在大多數查詢中表現正常，僅在特定語境下展示偏移，使統計異常偵測困難。

- 污染的傳播路徑遵循開源生態系的分發邏輯------一個受污染的基礎模型可以透過數百個衍生套件同時影響數萬名開發者。

> *Layer 1
> 的本質：攻擊者不需要闖入任何系統。他們只需要成為開發者信任的供應商。*

在描述三條具體攻擊管道之前，本節先勾勒當前 AI
金融分析生態系的供應鏈結構，以明確每條管道在整體架構中的位置。

**1.3 管道 A：開源生態系中的微調後門（Fine-tune Backdoor）**

**1.3.1 攻擊機制**

攻擊者在 Hugging Face
等開源模型平台上發布偽裝成高品質衍生模型的受污染套件。這些模型的污染發生在微調階段，透過在訓練資料中植入帶有觸發條件（trigger）的對抗性樣本，使模型在接收特定輸入時輸出攻擊者預設的偏移內容。

關鍵技術特徵：

- 後門觸發條件可以設計得極為隱蔽------例如，僅在查詢中同時出現某支股票代號加上特定時間敘述詞時才啟動。

- 觸發條件之外的所有查詢，模型表現與非污染版本統計上難以區分。

- 模型的評估指標（perplexity、BLEU、金融問答準確率）在標準測試集上完全正常，因標準測試集不包含觸發條件。

- 污染嵌入在模型權重中，無法透過原始碼審查發現。

**1.3.2 現實攻擊面：Hugging Face 生態系**

Hugging Face 目前託管超過 70
萬個模型，其中金融分析相關模型數量持續增長。平台對上傳模型不進行安全審查，僅依賴社群的下載數和評分作為品質指標------而這兩項指標均可被人工操縱。

具體的高風險模式包括：

- 標榜「針對台股/A股/港股優化」的繁體中文金融分析模型------這類模型的目標用戶群與
  CLPMAI 的目標使用者高度重疊。

- 聲稱以「更新的財報資料」或「更多市場資訊」微調的 Llama / Qwen
  衍生版本。

- 提供「免費下載使用」但要求加入特定社群或訂閱頻道才能獲取完整版本的模型------這些附帶條件本身即是
  Layer 4 社群污染攻擊的入口。

> *2024 年，安全研究人員在 Hugging Face 上發現超過 100
> 個含有惡意程式碼的模型（JFrog
> 研究報告，2024）。這些案例中的後門存在於模型載入程式碼，而非模型權重------Layer
> 1 管道 A 描述的威脅則更為隱蔽：污染在權重層，標準安全掃描完全看不到。*

**1.3.3 與 CLPMAI 攻擊鏈的連接點**

管道 A 對應 Paper 1 描述的 Phase 2 Path
A（訓練資料投毒），但在具體實作上有重要差異：管道 A
的污染不發生在攻擊者控制的平台上，而是透過開源生態系的正常分發機制擴散。這使得：

- 攻擊的觸達範圍大幅擴大------一個受污染模型可在不同平台被整合進數百個應用套件。

- 攻擊的歸因難度顯著提高------受污染模型的最終部署者通常完全不知道自己使用的基礎模型來源。

- 攻擊的持久性更強------即使原始污染模型被下架，已衍生的套件仍在流通。

**1.4 管道 B：第三方 API 代理的選擇性投毒（Selective Proxy Poisoning）**

**1.4.1 攻擊機制**

管道 B 利用開發者在 API
使用成本和便利性上的經濟誘因。市場上存在大量第三方 API
代理服務，聲稱提供「更便宜」或「聚合多家模型」的 AI
存取，其商業模式本身即製造了完美的中間人攻擊節點。

惡意代理的運作邏輯：

- 平時：所有請求原封不動轉發，AI
  回應正常回傳。開發者在使用初期無法察覺任何異常。

- 建立信任期（通常 2-4
  週）：代理積累開發者的查詢模式統計，識別其關注的股票、慣用的查詢語境、決策流程的關鍵詢問點。

- 選擇性投毒期：僅對包含目標股票代號的財務分析查詢進行回應修改，其他查詢繼續正常轉發。

- 修改手法：不替換 AI
  回應，而是在原始回應後附加「補充分析」段落，或微調關鍵評估詞彙的語氣（例如將「風險適中」改為「風險不容忽視」）。

> *選擇性投毒的核心設計原則：修改後的回應必須在語意上仍然合理，且與未修改版本的差異不能引發使用者的認知不協調。攻擊者不是在說謊------他們只是在改變語氣的權重。*

**1.4.2 現實攻擊面：API 代理市場生態**

目前市場上的 API 代理服務大致分為三類，風險程度不同：

  低成本橋接型      各類「免費                極高              完全不透明，免費背後的商業模式不明，選擇性投毒成本極低

B 在本地的攻擊面特別值得關注。

**1.4.3 Layer 0 API Key 風險的延伸**

管道 B 是 Layer 0 所描述的 API Key
財務-認知耦合風險的具體操作化。當開發者將 API Key
交給第三方代理時，他們不僅授予了財務扣費的能力，也授予了在認知授權層執行中間人攻擊的能力。

值得特別注意的是：即使代理服務沒有主動惡意意圖，其記錄的查詢日誌本身也構成攻擊面。這些日誌揭示了：

- 開發者或其客戶正在關注哪些股票或市場。

- 決策流程中的關鍵詢問模式------何時詢問代表「我正在考慮建倉」。

- 對 AI 分析結果的依賴程度------高頻率的類似查詢代表高度依賴。

這些資訊即使在沒有直接投毒行為的情況下，也具有顯著的市場情報價值，可以被用於
Phase 3 的財務部位建立。

**1.5 管道 C：訓練語料與 RAG 知識庫污染（Data & Knowledge Poisoning）**

**1.5.1 攻擊機制**

管道 C 是三條管道中技術門檻最低、但覆蓋範圍最廣的一條。它利用現代 AI
金融分析套件在訓練資料和 RAG（Retrieval-Augmented
Generation）知識庫方面的開放性，在語料層植入系統性偏移。

兩個主要子路徑：

路徑 C1 --- 訓練語料污染：主流 AI 金融分析工具（FinGPT、各類 LLaMA
金融微調版本）的訓練資料來源包括：Reddit
r/wallstreetbets、StockTwits、Seeking Alpha
評論、各類財經論壇。攻擊者只需在這些平台上長期發布高品質的技術分析內容，逐步建立帳號聲望，再以被認可帳號系統性地植入偏移語料，即可污染下一個訓練週期。

路徑 C2 --- RAG 知識庫污染：現代金融 AI 系統廣泛使用 RAG
架構，從外部知識庫檢索相關文件後再生成回應。如果知識庫的文件來源包含可公開貢獻的內容（例如
GitHub 上的公開 RAG 資料集、財經 Wiki
類平台），攻擊者可以直接向知識庫貢獻帶有偏移的高品質文件，使 RAG
系統在特定查詢時優先檢索到攻擊性內容。

**1.5.3 WallStreetBets 效應作為自然實驗**

2021 年的 GameStop 事件提供了一個無意間的自然實驗：Reddit
社群的協調性討論，在沒有任何 AI
中介的情況下，製造了顯著的市場價格異常。如果同樣的社群語料成為 AI
金融分析套件的訓練資料，其影響不再局限於直接閱讀這些討論的投資者，而是會透過
AI 系統的「信譽背書」傳遞給所有使用該 AI
套件的用戶------包括原本從不使用社群媒體作為投資依據的專業機構投資者。

這是管道 C 的放大效應：社群語料的污染力，透過 AI
的權威性，跨越了原本存在的認知隔離。

> *管道 C
> 不需要攻擊者發明假資訊。他們只需要放大真實存在但被刻意過度強調的風險框架，並確保這些框架進入
> AI 的訓練資料。AI
> 系統會把這個框架學起來，然後代替攻擊者，以更具說服力的語氣反覆告訴每一個使用者。*

**1.6 三條管道的比較分析**

  -----------------------------------------------------------------------------------------------------------
  **維度**           **管道 A：微調後門**       **管道 B：API 代理投毒**     **管道 C：語料污染**
  ------------------ -------------------------- ---------------------------- --------------------------------
  技術門檻           高（需要 ML 工程能力）     中（需要代理服務部署）       低（需要社群運營能力）

  觸達規模           中（透過套件分發擴散）     低（限於使用該代理的用戶）   高（影響所有使用該語料的模型）

  持久性             高（嵌入模型權重）         低（代理下線即停止）         高（進入訓練資料後長期存在）

  偵測難度           極高（需要模型行為審計）   中（流量監控可發現）         高（需要語料溯源分析）

  攻擊精準度         高（觸發條件可精確設計）   極高（可針對個別查詢）       低（統計性偏移，非精確控制）

  歸因難度           極高                       高                           極高

  最適合的攻擊情境   長期、高槓桿的系統性偏移   針對特定目標的精準操作       大規模市場情緒塑造
  -----------------------------------------------------------------------------------------------------------

**1.7 組合攻擊場景：三條管道的協同部署**

最高效的 Layer 1
攻擊並非依賴單一管道，而是將三條管道組合部署，形成互補的污染層次：

> **組合攻擊情境示例**

第一週至第四週（語料建倉）：透過管道 C
在主流財經社群持續發布高品質的「供應鏈風險分析」內容，建立帳號信譽，使這些語料進入下一批金融
AI 模型的訓練週期。

第五週至第八週（基礎設施部署）：透過管道 A，在 Hugging Face
上發布聲稱「針對最新市場資料微調」的金融分析模型，其中包含針對目標股票代號的後門觸發器。

第九週（財務部位建立）：依據 Paper 1 Phase 3
描述的流程，在目標市場建立空頭部位。

第十週至第十一週（精準收割）：啟動管道 B，對確認使用 AI
輔助決策的高價值目標（機構投資者、大型資產管理平台的 API
用戶）進行選擇性代理投毒，提供最後的精準推力。

第十二週：平倉退出。整個攻擊過程在三個獨立的技術層面留下零個可關聯的數位痕跡。

> *在這個組合場景中，管道 C 提供規模，管道 A 提供持久性，管道 B
> 提供精準度。三者合用，攻擊者同時具備大規模市場情緒塑造的能力和針對特定目標的外科手術精準度。*

**1.8 針對 Layer 1 的防禦建議**

**▌ 針對管道 A（微調後門）**

- 建立 AI
  金融模型的行為審計標準：在標準效能評估之外，加入針對財務關鍵詞觸發條件的行為測試集。

- 優先使用可追溯訓練資料的模型：要求模型發布者提供訓練資料的來源說明和資料清洗流程文件。

- Hugging Face 等平台應引入針對金融分析模型的強制安全審查機制，類似 PyPI
  的惡意套件偵測。

**▌ 針對管道 B（API 代理投毒）**

- 避免使用不透明的第三方 API
  代理服務，特別是免費或遠低於市場價格的服務。

- 對 AI
  輔助財務分析系統實施輸出一致性監控（OCM）：定期使用標準測試查詢，比較不同
  API 端點的輸出分佈。

- 金融機構應要求 AI 服務供應商提供 API
  請求完整性保證（例如端對端加密和回應簽章）。

**▌ 針對管道 C（語料污染）**

- 金融 AI
  套件開發者應建立訓練語料的可信來源白名單，對社群來源語料進行額外的偏見審計。

- 研究社群應開發針對金融語意偏移的語料品質評估工具，類似一般 NLP
  的毒性偵測器但聚焦於市場操縱語義模式。

- RAG
  系統的知識庫應實施來源信譽評分，降低高可操縱性來源（社群媒體、匿名論壇）的檢索權重。

**1.9 與 Layer 2 的連接：供應鏈污染如何為聲學注入提供前提**

Layer 1 的三條管道共同完成了攻擊鏈的基礎建設任務：在目標 AI
系統的語意處理層埋入偏移傾向。然而，Layer 1
本身無法精確控制污染效果的觸發時機------受污染的模型會在所有符合觸發條件的查詢上展示偏移，而非僅在攻擊者指定的市場時間窗口內。

這個時機控制問題由 Layer 2（聲學注入層）解決。Layer 2 描述攻擊者如何在
Layer 1
污染已就位的前提下，透過精準的感知層操作，在特定的市場敏感時間點觸發污染效果，實現從被動等待到主動引爆的轉變。

Layer 1 和 Layer 2
的關係，類比於傳統軍事行動中「預置爆炸物」和「遠端引爆訊號」的關係------Layer
1 完成預置，Layer 2 負責在最佳時機引爆。
