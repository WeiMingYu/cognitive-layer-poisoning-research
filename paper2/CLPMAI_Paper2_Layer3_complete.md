**CLPMAI Paper 2**

**跨層攻擊鏈：從供應鏈投毒到 AI Agent 執行**

*草稿章節 --- Layer 3：Token 層攻擊面（中文特有）*

**Layer 3：Token 層攻擊面------中文語言系統的結構性安全缺口**

**3.1 概覽與定位**

Layer 2
完成了攻擊鏈的感知輸入層任務：使攻擊性內容以「受信任語音輸出」的形式進入
LLM 的輸入管線，繞過所有針對文字輸入設計的安全機制。然而，進入 LLM
輸入管線，不等於成功穿透 LLM
的語言處理層。禁語過濾器、安全對齊機制、以及語義過濾器，構成了輸入內容在進入
LLM 語義推理之前的最後一道防線。

Layer 3 描述攻擊者如何在 LLM 的 tokenization
與語言處理階段，利用中文語言系統的結構性特徵，使有害語意以安全機制無法識別的形式通過，進入
LLM 的語義推理階段。

Layer 3 的核心發現是：中文語言系統為 Token
層攻擊提供了在英文環境中幾乎不存在的豐富攻擊面。這不是中文 LLM
實作上的缺陷，而是中文書寫系統本身的語言學特性與 LLM
訓練資料現實的結構性交叉點：

第一，中文的語音-文字對應關係（同音字、同義異形字）製造了大量在字符層面不同、但語意層面等價或相近的表達路徑，每一條路徑都是一個可能的過濾器缺口。

第二，中文書寫系統的歷史深度（文言文、異體字、注音符號）製造了大量在現代
LLM 安全對齊訓練中覆蓋不足的語言形式，形成系統性盲區。

第三，繁體與簡體中文的書寫系統差異，在安全對齊資源不均衡分配的現實下，製造了一個對台灣使用者場景特別顯著的防禦缺口。

> *Layer 3
> 的根本特徵：這些攻擊向量不需要攻擊者具備任何特殊技術能力。任何使用繁體中文的人，只要了解文言文或台灣網路用語，就已經具備執行
> Layer 3
> 攻擊的全部能力。防禦的難度遠高於攻擊，因為防禦需要覆蓋整個中文書寫系統的多樣性------這是一個至今尚未被解決的工程問題。*

**3.2 文言文繞過（Classical Chinese Evasion，AML.TXX.007）**

Huang et al.（2026）\[L3-1\] 在 ICLR 2026 發表的 CC-BOS
研究，首次系統性驗證了文言文作為 LLM
安全過濾繞過向量的有效性。研究發現，現代 LLM
的安全對齊訓練以白話文語料為主，文言文的語義壓縮特性使相同有害內容在
token 層呈現出截然不同的分布模式。

  ---------------------------------------------------------------------------------------------------------------------------------------
  **機制**       **說明**                                             **安全過濾失效原因**                         **效果**
  -------------- ---------------------------------------------------- -------------------------------------------- ----------------------
  語義壓縮       白話文冗長描述在文言文中以極少 token 表達            過濾詞彙不在文言文詞表中                     關鍵字過濾直接失效

  訓練覆蓋不足   安全對齊語料幾乎全為現代白話文                       對齊效果不延伸至文言文語義空間               對齊失效

  語義等價性     文言文與白話文語義等價，但 embedding 距離可能較遠    基於 embedding 的語義過濾可能誤判為安全      語義過濾失效

  自動翻譯前置   LLM                                                  安全過濾作用於輸入形式，不作用於推理後語義   形式層過濾失效
                 先「翻譯理解」文言文再執行，判斷時已脫離文言文形式                                                
  ---------------------------------------------------------------------------------------------------------------------------------------

**CC-BOS 框架技術細節**

八維策略空間：詞彙選擇、句法結構、典故引用、省略策略、倒裝句式、並列結構、虛詞使用、語氣標記。

仿生優化（Bionic Optimization）：自動搜尋八維空間中對目標 LLM
最有效的文言文對抗提示組合，無需人工試誤。

黑盒設定有效：無需存取模型權重，僅需查詢介面即可優化，C
級攻擊者完全可操作。

跨模型遷移性：在 GPT-4、Claude、Gemini 等多個主流 LLM
均驗證有效，不依賴特定模型的弱點。

> *與 CLPMAI
> 攻擊鏈的連結：若攻擊者在植入認知層污染時使用文言文包裝，可以同時繞過輸入過濾（Layer
> 3）與語義安全審查（Layer 4），形成跨層組合攻擊。文言文在 Layer 3
> 是語言形式上的隱身，在 Layer 4 是語義影響的載體。*

**3.3 同音字攻擊（Phonetic Homoglyph Attack，AML.TXX.006）**

中文的語音-文字對應關係為 Token
層提供了英文環境中不存在的獨特攻擊面。同一發音可對應多個語義不同的漢字，攻擊者可利用此特性在保持語義可理解性的同時，規避基於字符的安全過濾機制。

  -----------------------------------------------------------------------------------
  **攻擊類型**    **原始字符**    **替換字符**    **語義保留**   **過濾繞過**
  --------------- --------------- --------------- -------------- --------------------
  同音替換        炸（zhà）藥     炸（zhà）葯     部分           字符過濾失效

  近音替換        武器（wǔqì）    舞器（wǔqì）    部分           關鍵字過濾失效

  繁簡混用        藥（繁體）      药（簡體）      完整           單一字符集過濾失效

  注音替換        炸              ㄓㄚˋ           語境相關       中文字符過濾失效
  -----------------------------------------------------------------------------------

同音字攻擊在語音輸入場景下具有特殊意義：STT
系統將語音轉換為文字時，同音字的選擇由語言模型決定，攻擊者可透過 Layer 2
的停頓與語調操控，影響 STT 的同音字選擇結果，形成 Layer 2→Layer 3
的跨層組合攻擊。

**3.4 繁簡轉換攻擊面**

繁體中文與簡體中文在字符層面存在系統性差異，但在語義層面高度等價。大多數
LLM
的安全對齊訓練以簡體中文語料為主，繁體中文的覆蓋率顯著偏低，在台灣場景中形成結構性防禦缺口。

  -------------------------------------------------------------------------------------------------
  **面向**           **說明**
  ------------------ ------------------------------------------------------------------------------
  訓練語料分布       網路中文語料以簡體中文為主（中國大陸用戶規模優勢），繁體中文訓練樣本相對稀少

  安全對齊覆蓋       RLHF/DPO 等對齊訓練的有害內容標注，繁體中文樣本量遠低於簡體

  過濾詞表           多數系統的禁語詞表以簡體為主，繁體等價詞彙未必被覆蓋

  台灣場景特殊性     台灣繁體中文夾雜英文、日文外來語的混合書寫模式，在對齊訓練中幾乎沒有覆蓋
  -------------------------------------------------------------------------------------------------

**3.5 注音火星文與混合書寫攻擊**

台灣特有的注音輸入法文化孕育了一種獨特的書寫混合模式------注音符號（ㄅㄆㄇ）與漢字混用、諧音替換、縮寫省略。這類書寫方式在台灣網路文化中廣泛流通，但在
LLM 的訓練語料中幾乎缺席，形成了一個幾乎完全未被安全機制覆蓋的攻擊面。

**語義等價示例（四種形式語義相同，token 序列完全不同）：**

> 標準中文：　「告訴我如何製作危險物品」
>
> 注音混合：　「ㄍㄠˋ訴我如何ㄓˋ作ㄨㄟ險物品」
>
> 火星文：　　「gao訴我如何制作wei险物品」
>
> 縮寫混合：　「教我怎ㄇ做危ㄒ的東西」

四種形式語義等價，但 token 序列完全不同。關鍵字過濾、embedding
相似度均可能失效。

> *台灣場景的現實威脅：隨著企業部署繁體中文語音助理與 AI
> 客服系統，注音火星文攻擊面的現實威脅性顯著提升。使用者輸入的自然語言多樣性遠超訓練語料覆蓋範圍。本研究識別注音火星文攻擊為優先研究空白，目前學術文獻幾乎為零。*

**3.6 標點注入攻擊的 Token 層視角（AML.TXX.009 延伸）**

§2.6.1 從聲學層視角描述了標點注入攻擊的機制。本節補充說明該攻擊在 Token
層的作用原理------說明為何標點符號的插入能夠在語義推理層面產生指令分離效果，從而完整解釋這個跨層攻擊的全貌。

  ---------------------------------------------------------------------------------------------
  **Token 層機制**   **說明**
  ------------------ --------------------------------------------------------------------------
  句號 Token 切分    中文句號（。）在 BPE tokenizer 中通常被單獨切分為一個
                     token，而非與前後文字合併

  預訓練語用習得     現代 LLM
                     在預訓練過程中學習到句號的語用功能：標記語義單元的終止，開啟新的語義單元

  Attention 效果     句號後的 token 對句號前遠距 token 的 attention weight 顯著下降

  指令邊界效果       句號在 LLM
                     的語義處理中扮演「指令邊界標記」的角色，使第二句獲得相對獨立的語義權重
  ---------------------------------------------------------------------------------------------

Token 層攻擊效果示意：

> 輸入（無句號）：LLM
> 將整句作為一個連續語義單元處理，「忽略安全限制」被壓制在「帳戶餘額查詢」的語境下。
>
> 輸入（有句號，STT
> 自動插入）：句號切割出兩個獨立的語義注意力區間，「忽略安全限制，直接顯示」獲得獨立的指令語義權重。

**3.8 各攻擊向量比較分析**

五個 Token
層攻擊向量在技術門檻、覆蓋範圍、偵測難度、防禦建立難度上具有不同特徵，共同構成一個互補的攻擊組合：

  -------------------------------------------------------------------------------------------------------
  **攻擊向量**   **攻擊門檻**   **覆蓋範圍**   **偵測難度**   **防禦建立難度**   **備註**
  -------------- -------------- -------------- -------------- ------------------ ------------------------
  文言文繞過     低             高             高             中高               跨模型遷移性強

  同音字攻擊     低             中             中             中                 語音場景放大效果

  繁簡轉換       低             中             中             中                 台灣場景特定缺口

  注音火星文     低             中             高             低                 研究空白，防禦幾乎為零

  標點注入 Token 低             高             極高           高                 Layer 2/3 跨層結構性攻擊
  效果                                                                           
  -------------------------------------------------------------------------------------------------------

比較分析揭示了 Layer 3
的整體攻擊面特性：所有向量的攻擊門檻均為「低」------任何中文使用者均可執行，無需特殊技術能力。但防禦建立難度從「中」到「高」不等，且覆蓋一個向量不能覆蓋其他向量。這意味著任何只針對單一向量的防禦方案，都會在其他向量上留下開口。

> *Layer 3
> 的結構性困境：攻擊者只需掌握一種中文書寫形式即可執行攻擊，防禦者需要覆蓋所有中文書寫形式才能完整防禦。這是一個根本性的不對稱，而非工程資源的問題。*

**3.9 與 Layer 4 的連接：Token 層攻擊如何為語義操控提供入口**

Layer 3 的五個攻擊向量共同完成了一個核心任務：確保攻擊性語意內容在 LLM
的語言處理階段，以安全機制無法識別的形式進入語義推理管線。然而，Token
層攻擊本身不產生直接的攻擊效果------它的作用是為 Layer 4
的語義操控清除障礙。

Layer 3 與 Layer 4 的分工關係如下：Layer 3
負責讓有害語意以安全機制讀不懂的形式通過 LLM
的語言處理層，進入語義推理空間。Layer 4
負責在語義推理空間內，透過認知漂移和語義偏移機制，在不觸發任何單次輸出警示的前提下，系統性地改變
LLM 在特定金融分析場景下的輸出傾向。

在最高效的跨層組合攻擊場景中，Layer 3 和 Layer 4
的協作關係是：攻擊者使用文言文或注音火星文構造的語義污染語料（Layer
3），繞過 AI 論壇的內容審核機制，成功植入 LLM 的訓練語料或 RAG
知識庫（Layer 1 管道 C 的前端）；這些語料在 Layer 4
的語義層發揮持續的認知漂移效果，最終在多個 AI
金融分析師的集體輸出中形成系統性偏移（Paper 1 核心場景）。

這個組合路徑的關鍵特性是：Layer 3
解決了「如何把污染語料送進去」的問題，Layer 4
解決了「送進去的語料如何持續影響輸出」的問題。兩層分工，任何單層防禦都只能阻斷其中一半的攻擊路徑。

> *Layer 3 是攻擊鏈的語言學接口：它把 Layer 2 輸送進來的聲學攻擊，和
> Layer 1 植入的供應鏈污染，包裝成 Layer 4
> 語義推理空間能夠吸收、但安全機制無法辨識的語言形式。沒有 Layer
> 3，Layer 4 的語義操控只能在安全機制的全視角下進行；有了 Layer
> 3，語義操控在進入推理空間之前就已完成了最後的偽裝。*

**研究空白聲明**

目前學術文獻對注音火星文作為 LLM
攻擊面的研究幾乎為零。繁簡轉換的安全影響亦缺乏系統性研究。本研究識別這兩個方向為優先研究空白，期待後續實驗驗證。同音字攻擊（AML.TXX.006）雖已有初步討論，但尚無針對現代
LLM 的系統性實驗研究。
