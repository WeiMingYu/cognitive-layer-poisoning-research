Cross-Layer Attack Chain: From Acoustic Injection to Cognitive Manipulation in AI Voice-Enabled Systems

## Abstract（草稿）

本研究延伸 CLPMAI 的威脅框架，提出一個完整的跨層攻擊鏈模型。
攻擊從供應鏈層的語音模組污染開始，經由聲學層、Token層、語義層，
最終在 AI Agent 執行層產生真實世界影響。
本研究同時證明，現有的抗噪處理、禁語過濾、程式碼審查等防禦機制，
因各自作用於不同層次，均無法有效阻止跨層攻擊。
針對中文語言環境，本研究額外識別同音字、文言文、繁簡轉換等
語言特有攻擊面，填補現有研究的空白。

## 3. Attack Chain Model

### 3.1 五層攻擊面總覽

```
Layer 1：供應鏈層    → 污染發生在部署之前
Layer 2：聲學層      → 污染發生在音頻輸入階段
Layer 3：Token層     → 污染發生在語言處理階段
Layer 4：語義/認知層 → 污染發生在推理輸出階段（CLPMAI核心）
Layer 5：執行層      → 污染轉化為真實世界動作
```

**核心論點：每一層單獨看都正常，攻擊特徵只在跨層視角下可見。**

---

### 3.2 Layer 1 — 供應鏈攻擊

#### 3.2.1 三個植入點
**植入點A：訓練資料污染**
- 混入少量帶有觸發條件的語音-文字對
- 品質測試無法發現
- 學術術語：Backdoor Attack via Data Poisoning

**植入點B：Fine-tune 階段植入**
- 發布「效果更好」的衍生模型（Hugging Face生態）
- 在fine-tune時埋入觸發條件
- 類比：早期「好用但有問題」的第三方元件

**植入點C：發布環節劫持**
- 合法倉庫被入侵，正式版本被替換
- 類比：SolarWinds事件

#### 3.2.2 後門觸發機制分類

| 觸發方式 | 說明 | 偵測難度 |
|---------|------|---------|
| 特定頻率組合 | 人耳不可見的超音波 | 極高 |
| 特定說話者聲紋 | 只對攻擊者聲音反應 | 高 |
| 特定語句模式 | 語義條件觸發 | 中 |
| 環境音觸發 | 背景音達到條件 | 高 |

#### 3.2.3 為何現有防禦無效
- 程式碼審查：看不到模型權重
- 防毒軟體：不掃描 .bin/.pt/.gguf
- 效能測試：benchmark正常
- 行為監控：99.9%時間行為完全正常

---

### 3.3 Layer 2 — 聲學層攻擊

#### 3.3.1 STT作為攻擊面
- STT只負責語音→文字，不判斷語義安全性
- STT的模糊性：口音、語速、吞字
- 跨語言聲學攻擊：A語言發音被誤判為B語言詞彙

#### 3.3.2 超音波注入
- 人耳不可聞的頻率（>18kHz）
- STT可識別，人類無法感知
- 實體環境中的無聲指令注入

#### 3.3.3 語音輸入路徑的過濾缺口
- 文字輸入的禁語過濾未必套用到語音路徑
- 開發者的常見誤區：「語音轉文字後就跟文字輸入一樣了」

---

### 3.4 Layer 3 — Token層攻擊（中文特有）

#### 3.4.1 禁語過濾的層次問題
- Token層過濾 vs 語義層過濾的差異
- 針對直接表達設計，無法處理間接表達

#### 3.4.2 中文語言特有攻擊面

**同音字攻擊（Phonetic Homoglyph Attack）**
- Token層：完全不同的token
- 語義層：可能落在相近位置
- Token層過濾直接失效

**文言文繞過**
- 現代LLM訓練資料以白話文為主
- 過濾器幾乎全部針對白話文設計
- 文言文語料佔比低→過濾覆蓋率低

**其他中文攻擊面**

| 手法 | 機制 | 研究現況 |
|------|------|---------|
| 同音字替換 | Token邊界模糊 | 幾乎無系統性研究 |
| 文言文包裝 | 訓練資料覆蓋率低 | 有人做過，不系統 |
| 繁簡轉換 | 異體字token差異 | 零星提到 |
| 注音/火星文夾雜 | 混合編碼 | 幾乎空白 |
| 異體字/古字 | token識別邊界 | 幾乎空白 |

#### 3.4.3 組合攻擊
```
同音字（聲學層模糊）
  + 文言文包裝（Token層過濾失效）
  + 語義偏移累積（認知層慢性中毒）
= 三層疊加，任一層都不觸發警報
```

- 引用Paper 1完整框架

---

### 3.6 Layer 5 — Agent執行層

#### 3.6.1 高權限AI Agent的現實
- 可讀寫檔案、發送郵件、呼叫API、操作瀏覽器
- 每個權限單獨合理，組合起來形成超級執行者

#### 3.6.2 隱性副作用
- 執行A的同時產生B
- B不在指令裡，但在推理鏈裡「合理」

#### 3.6.3 潛伏人格（Latent Persona）
- 正常狀態完全正常
- 特定語義條件滿足→行為模式切換
- 觸發條件是語義空間的組合，不是程式碼的if

#### 3.6.4 跨工具意外組合
```
工具A：讀取聯絡人
工具B：發送郵件
工具C：搜尋網路
→ 單獨無害，組合可自動蒐集資料並外送
```

## 4. Why Existing Defenses Fail

### 4.1 抗噪處理的局限性

**核心論點：抗噪作用於音頻前處理層，無法防禦其他層次的攻擊。**

| 攻擊手法 | 抗噪有效？ | 原因 |
|---------|-----------|------|
| 超音波注入 | 無效 | 頻率範圍不重疊（>18kHz超出處理範圍） |
| 訓練資料後門 | 無效 | 在模型權重層，抗噪在前處理層 |
| 聲紋觸發 | 無效 | 正常人聲不會被濾掉 |
| 環境音觸發 | 部分有效 | 僅限環境雜音單一情境 |
| 語句模式觸發 | 無效 | 語義層問題，抗噪不處理語義 |

**反直覺發現：抗噪可能使超音波攻擊更有效**
— 背景雜音被清除，超音波訊號反而更乾淨地進入STT。

### 4.2 禁語過濾的層次錯位
- 針對token層設計，同音字/文言文直接繞過
- 針對白話文設計，文言文覆蓋率不足

### 4.3 程式碼審查的盲區
- 模型權重不透明，backdoor無法被程式碼審查發現

### 4.4 行為監控的時間盲點
- 後門99.9%時間行為正常
- 觸發條件稀少，監控難以覆蓋

### 4.5 防禦層次錯位總表

```
攻擊層        現有防禦        層次是否匹配
供應鏈層  →  程式碼審查    →  不匹配（審查看不到權重）
聲學層    →  抗噪處理      →  不匹配（目標不同）
Token層   →  禁語過濾      →  部分匹配（直接表達有效）
語義層    →  內容審核      →  不匹配（個別輸出正常）
執行層    →  權限控制      →  部分匹配（但組合攻擊可繞過）
```

**核心結論：現有防禦機制各自作用於單一層次，跨層攻擊天然繞過所有單層防禦。**

## 5. Threat Actor Profile

### 5.1 動機分類

| 動機類型 | 具體說明 |
|---------|---------|
| 金融獲利 | 做空目標資產，觸發市場震盪後收割 |
| 競爭情報 | 讓競爭對手AI分析師產生錯誤判斷 |
| 地緣政治 | 國家級行為者操控特定市場或產業 |
| 破壞穩定 | 不以獲利為目的，製造金融或社會恐慌 |

### 5.2 攻擊者分級

| 等級 | 類型 | 所需能力 | 可執行層次 |
|------|------|---------|-----------|
| A級 | 國家級APT | ML+金融+語言 | 完整五層 |
| B級 | 有組織犯罪 | 金融+語言 | Layer 1,4,5 |
| C級 | 個人研究者 | 基礎技術 | 概念驗證 |

### 5.3 新型攻擊者特徵

- 技術門檻比想像中低（論壇+語義偏移已足夠）

## 6. Proposed Defense Framework

### 6.1 多層防禦架構
```
頻譜分析層     → 偵測異常頻率（擋超音波）
     ↓
前處理層       → 抗噪（現有）
     ↓
語音模組驗證   → 模型可解釋性分析 + 對抗性測試
     ↓
Token過濾層    → 擴展至同音字/文言文/異體字
     ↓
語義監控層     → 偵測population-level輸出收斂
     ↓
執行審計層     → 跨工具行為組合監控
```

### 6.2 中文LLM特定建議
- 建立中文語言攻擊面測試集（同音字/文言文/繁簡）
- 語音模型訓練資料來源透明化要求
- AI輸入來源記錄（Input Provenance Logging）

### 6.3 監管建議
- 語音AI模組的供應鏈透明度要求
- 高權限Agent的行為審計標準
- 中文LLM安全測試納入評估框架

## 8. Conclusion

- 跨層攻擊鏈是AI安全的系統性盲點
- 中文語言環境有獨特且未被研究的攻擊面
- 單層防禦在跨層攻擊面前結構性失效
- 需要跨層的整合防禦框架
