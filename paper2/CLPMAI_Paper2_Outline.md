# Abstract

本研究延伸 CLPMAI（Paper
1）的威脅框架，提出一個完整的跨層攻擊鏈模型。攻擊從供應鏈層的語音模組污染開始，經由聲學層、Token
層、語義層，最終在 AI Agent 執行層產生真實世界影響。

本研究同時證明，現有的抗噪處理、禁語過濾、程式碼審查等防禦機制，因各自作用於不同層次，均無法有效阻止跨層攻擊。針對中文語言環境，本研究額外識別同音字、文言文、繁簡轉換等語言特有攻擊面，填補現有研究的空白。

本文並提出對應的多層防禦架構，以及五個新增 MITRE ATLAS
技術提案（AML.TXX.004 至 AML.TXX.008）。

# 1. Introduction

## 1.1 研究動機

- CLPMAI（Paper 1）建立了認知層攻擊的基礎框架，以 AI
  金融市場操控為核心案例

- 真實世界的攻擊不會只在單一層發生------供應鏈、聲學、語言、認知、執行各層均存在獨立攻擊面

- 現有研究缺乏跨層視角的系統性威脅模型

- 中文語言環境的 AI 安全攻擊面幾乎完全空白

## 1.2 研究貢獻

- 提出五層攻擊鏈模型（供應鏈 → 聲學 → Token → 語義 → 執行）

- 識別中文語言環境特有攻擊面：同音字、文言文、繁簡轉換、注音火星文

- 正式定義「防禦層次錯位」問題：現有防禦作用於單層，無法對抗跨層攻擊

- 提出多層整合防禦框架

- 新增五個 MITRE ATLAS 技術提案（AML.TXX.004--008）

## 2.3 核心論點：層次孤立性

攻擊鏈的根本危險在於其「層次孤立性」------每一個層次的防禦機制只能看到自己負責的那一層，而攻擊的惡意性只有在跨層視角下才可見。

  ------------------------------------------------------------------------
  **防禦機制**       **作用層次**    **看不到的攻擊層**
  ------------------ --------------- -------------------------------------
  防火牆 / IDS       網路層          語義層、聲學層、供應鏈層

  抗噪處理           音頻前處理層    Token層、語義層、執行層

  禁語過濾           Token / 輸出層  聲學層、供應鏈層、跨層組合

  程式碼審查         原始碼層        模型權重層（後門不在程式碼裡）

  日誌審計           系統行為層      語義層（正常輸出不被記錄）

  市場監管           金融交易層      AI 推理過程（黑盒）
  ------------------------------------------------------------------------

# 3. 攻擊鏈詳細分析

## 3.1 Layer 1 --- 供應鏈攻擊

### 3.1.1 三個植入點

**植入點 A：訓練資料污染**

- 混入少量帶有觸發條件的語音-文字對

- 品質測試（benchmark）無法發現

- 學術術語：Backdoor Attack via Data Poisoning \[5\]\[6\]

**植入點 B：Fine-tune 階段植入**

- 發布「效果更好」的衍生模型（Hugging Face 生態）

- 在 fine-tune 時埋入觸發條件

- Bartolini et al. \[5\] 已證實 Whisper 在 fine-tune
  階段高度容易受此類攻擊

**植入點 C：發布環節劫持**

- 合法倉庫被入侵，正式版本被替換

- 類比：SolarWinds 事件

### 3.1.2 後門觸發機制分類

  -----------------------------------------------------------------------------------
  **觸發方式**             **說明**                               **偵測難度**
  ------------------------ -------------------------------------- -------------------
  特定頻率組合（超音波）   人耳不可見（\>18kHz），DolphinAttack   極高
                           \[4\] 已驗證                           

  特定說話者聲紋           只對攻擊者聲音反應                     高

  特定語句模式             語義條件觸發                           中

  環境音觸發               背景音達到特定條件，Hidden in Plain    高
                           Sound \[5\] 已驗證                     
  -----------------------------------------------------------------------------------

## 3.2 Layer 2 --- 聲學層攻擊

- STT 只負責語音→文字轉換，不判斷語義安全性

- 超音波注入：Zhang et al. DolphinAttack \[4\] 已在 Siri、Google
  Now、Alexa 等主流平台驗證

- 文字輸入的禁語過濾未必套用到語音輸入路徑

*開發者常見誤區：「語音轉文字後就跟文字輸入一樣了」*

## 3.3 Layer 3 --- Token 層攻擊（中文特有）

### 3.3.1 文言文繞過

Huang et al. \[3\]（ICLR 2026）首次系統性證明文言文在 LLM
安全過濾方面的繞過效果：

- 文言文的語義壓縮特性讓同樣含義的有害內容更難被關鍵字過濾偵測

- 提出 CC-BOS 框架：八維策略空間的仿生優化，自動生成文言文對抗提示

- 在黑盒設定下對多個主流 LLM 驗證有效

### 3.3.2 完整中文攻擊面分類

  ---------------------------------------------------------------------------------------
  **手法**         **機制**                                   **代表研究**
  ---------------- ------------------------------------------ ---------------------------
  文言文繞過       訓練資料覆蓋率低，過濾器幾乎全針對白話文   Huang et al. 2026 \[3\]

  同音字替換       Token 邊界模糊，語義空間可能相近           本研究提出（空白待填補）

  注音 /           混合編碼繞過單一語言過濾器                 本研究提出（空白待填補）

## 3.4 Layer 4/5 --- 認知層與執行層

- Layer 4 延伸 CLPMAI（Paper 1）完整框架：集體輸出偏移作為執行層觸發條件

- Layer 5 --- 潛伏人格（Latent
  Persona）：正常狀態完全正常，特定語義條件滿足後行為模式切換

- 高權限 Agent 的跨工具意外組合：讀取聯絡人 + 發送郵件 + 搜尋網路 =
  自動蒐集資料外送

# 4. 為什麼現有防禦失效

## 4.1 抗噪處理的局限性（反直覺發現）

  ------------------------------------------------------------------------------
  **攻擊手法**       **抗噪有效？**   **原因**
  ------------------ ---------------- ------------------------------------------
  超音波注入 \[4\]   無效             頻率範圍不重疊（\>18kHz 超出抗噪處理範圍）

  訓練資料後門 \[5\] 無效             在模型權重層，抗噪在前處理層，兩者無交集

  聲紋觸發           無效             正常人聲不會被抗噪濾掉

  環境音觸發 \[5\]   部分有效         僅限環境雜音此單一情境

  語句模式觸發       無效             語義層問題，抗噪不處理語義
  ------------------------------------------------------------------------------

**反直覺發現：抗噪可能使超音波攻擊更有效------背景雜音被清除後，超音波訊號反而更乾淨地進入
STT。**

## 4.2 防禦層次錯位總結

  ------------------------------------------------------------------------
  **攻擊層**      **現有防禦**    **層次是否匹配**
  --------------- --------------- ----------------------------------------
  供應鏈層        程式碼審查      不匹配（審查看不到模型權重）

  聲學層          抗噪處理        不匹配（作用目標不同）

  Token 層        禁語過濾        部分匹配（直接表達有效，繞過無效）

  語義層          內容審核        不匹配（個別輸出正常，集體才異常）

  執行層          權限控制        部分匹配（組合攻擊可繞過）
  ------------------------------------------------------------------------

**核心結論：現有防禦機制各自作用於單一層次，跨層攻擊天然繞過所有單層防禦。**
